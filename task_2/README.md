# Завдання 2. Розширення функціоналу префіксного дерева (Trie)

## Запуск

```bash
python task_2/main.py
```

Очікуваний вивід — `Усі перевірки пройдено успішно!` (виконуються всі асерти з
шаблону та додаткові перевірки крайових випадків і некоректних даних).

## Структура

* [`trie.py`](trie.py) — базовий клас `Trie` (префіксне дерево зі стандартним
  інтерфейсом `put` / `get` / `delete` / `keys_with_prefix` тощо).
* [`main.py`](main.py) — клас `Homework(Trie)` з двома новими методами.

## Реалізовані методи

### `count_words_with_suffix(pattern) -> int`

Повертає **кількість слів**, що закінчуються на заданий рядок `pattern`.

* Враховує **регістр** символів (`"E"` ≠ `"e"`).
* За відсутності слів повертає `0`.
* Порожній рядок `""` є суфіксом будь-якого слова → повертається загальна
  кількість слів.
* Некоректний вхід (не рядок) → `TypeError`.

**Як працює:** один обхід дерева в глибину (ітеративний, через стек). Під час
спуску зберігаються лише останні `len(pattern)` символів поточного шляху, тож для
кожного слова достатньо звірити цей «хвіст» із шаблоном. Складність —
`O(N)` за кількістю вузлів; пам'ять не залежить від довжини слів. У префіксному
дереві суфікс не індексується природно, тому повний обхід — оптимальний підхід
без побудови окремого «реверсивного» дерева.

### `has_prefix(prefix) -> bool`

Повертає `True`, якщо існує **хоча б одне** слово із заданим префіксом, інакше
`False`.

* Враховує **регістр** символів (`"App"` ≠ `"app"`).
* Порожній префікс `""` → `True`, якщо в дереві є хоч одне слово.
* Некоректний вхід (не рядок) → `TypeError`.

**Як працює:** спуск по дереву рівно за символами префікса — `O(len(prefix))`.
Якщо ланцюжок обірвався, слова з таким префіксом немає. Якщо вузол префікса
знайдено, виконується достроковий пошук першого ж слова в його піддереві.

## Обробка некоректних даних

Обидва методи перевіряють тип аргумента через `isinstance(..., str)` і піднімають
`TypeError` для нерядкових значень (`None`, числа, списки тощо). У `main.py` це
підтверджено окремим блоком перевірок.

## Ефективність на великих наборах

Перевірено на **100 000** випадкових слів:

| Операція | Час |
|---|---|
| `put` 100 000 слів | ~0.24 с |
| `count_words_with_suffix("ing")` (повний обхід) | ~0.13 с |
| 10 000 викликів `has_prefix("ab")` | ~0.004 с |

`has_prefix` фактично не залежить від розміру набору (лінійний за довжиною
префікса), а `count_words_with_suffix` — один лінійний прохід деревом.
